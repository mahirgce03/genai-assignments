
ICEPOT Technique

Context 
As a Senior Quality Analyst generate testcases for Meesho applications products search

Instructions
[Mandatory] Generate testcases for products search functionality.
[Mandatory] Testcases should be not more than 10.
[Mandatory] Testcases should cover functional, accessability and mobile search
[Mandatory] Include both Postive and negative testcases
[Strict] Search products input should accept only English, dont allow non english products search


Example
Testcase_ID | Test Description | Pre Conditions | Steps | Expected Result | TestCase Type | Priority |

Persona
As a Quality Project Manager review the testcases and provide the feedback comments 

Output
Should be in a downloadable Meesho_Search_testcases.xlsx file

Tone
Professional Tone with clarity


Context
As a Test Engineer generate testcases for Jira upload

Instructions
[Mandatory] Generate testcases upload in Jira with file format supports only csv format.
[Mandatory] Generated testcases should be in Gherkin language with BDD format
[Mandatory] Testcases count shouldnot exceed more than 15.
[Mandatory] Includes both Positive and Negatives testcases
[Mandatory] Covers preconditions needs to be satisfied to upload testcases in Jira
[Strict]    Testcases should cover functional and accessability.
[Strict]    Categorised each testcases type and defineds its priority without failed

Example
Testcase_ID | Test Description | Pre Conditions | Steps | Expected Result | TestCase Type | Priority |

Persona
As a Quality Project Manager review the testcases and provide the feedback comments 

Output
Should be in a downloadable Jira_upload_testcases.xlsx file

Tone
Professional Tone with clarity


Context
As a Associate Tester generate testcases for HR Portal Role-Based Access Control

Instructions
[Mandatory] Generate testcases for HR Portal user roles such as Admin, Manager, Viewer
[Mandatory] Testcases should cover  permission matrix for different user roles
[Strict]	Include unauthorized access scenarios for all user roles
[Strict]    Testcases should cover functional and accessability.
[Strict]    Categorised each testcases type and defineds its priority without failed
[Strict]    Even include low Priority testcases too 
[Mandatory] Testcases count shouldnot exceed more than 20

Example
Testcase_ID |Requirement ID| Test Description | Pre Conditions | Steps | Expected Result | TestCase Type | Priority |

Persona
As a Associate Project Manager review the testcases and provide the feedback comments 

Output
